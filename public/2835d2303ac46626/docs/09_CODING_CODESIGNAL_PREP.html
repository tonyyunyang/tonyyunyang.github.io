1. CodeSignal Format and Strategy

1A. General Coding Assessment (GCA)

What eBay likely sends for Stage 2: - 70 minutes, 4 questions, strictly increasing difficulty - Webcam proctoring, full-screen lock, keystroke logging - No AI tools, no external lookup, no screen switching - Python strongly recommended for ML roles (fastest to write, fewest ceremony) - New score scale (2026): 200–600 replacing the old 300–850

Question ladder:

Question Time budget Difficulty Typical content
Q1 5–8 min Easy Loops, string ops, basic array manipulation
Q2 20 min Medium Hash maps, binary search, simple BFS
Q3 25 min Medium-Hard DP, graphs, sliding window
Q4 remaining Hard Complex DP, graph optimization, advanced combinatorics

Score targets: - 475–525 = strong competitive candidate - 525+ = elite signal - Partial credit is real and meaningful: a brute-force Q4 attempt that passes 3/10 test cases is worth far more than a blank

Mental model for scoring: Questions are weighted by difficulty. A perfect Q1–Q3 with a brute-force Q4 attempt scores significantly higher than stopping after Q3. Never leave Q4 blank.

Some eBay tracks use this instead of (or after) the GCA. Structure:

Duration: 70–90 minutes, auto-timed modules (cannot go back)

1C. Time Management Strategy

For GCA: 1. Read all four problem statements in the first 3 minutes. Know what you're walking into. 2. Q1: solve in ≤8 min, check one edge case (empty input, single element, negatives) 3. Q2: pattern-match first (hashmap? two-pointer? BFS?). Code in ≤20 min. O(n) or O(n log n). 4. Q3: spend 5 min identifying the pattern before coding. Aim for O(n log n) or O(n²) if n is small. 5. Q4: with 12–15 min left, code a brute-force or partial solution. Any passing test cases = free points. Submit even if wrong.

For ML Assessment: 1. Burn through MCQs fast (Modules 1+2); aim for 1.5 min/question maximum. 2. Do not overthink probability questions; the answer is almost always Bayes or CLT. 3. For Module 3 coding: read the full question, plan on paper/scratch before typing. 4. Use sklearn when allowed (faster); only write from scratch if explicitly asked.

AI-native candidate's specific threat: You have built muscular memory for delegating to Claude Code. Your biggest risk is blank-screen anxiety when the autocomplete disappears. The antidote is daily cold-coding practice with zero IDE assistance — raw Python in a text editor or repl.it with autocomplete disabled.

Pattern 1: Arrays / Strings / Hashmaps

When: Frequency counting, duplicate detection, grouping, two-sum family, substring problems.

Core structures: dict, set, collections.Counter, collections.defaultdict

Two Sum (canonical hashmap problem):

def two_sum(nums, target):
    seen = {}  # value -> index
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:
            return [seen[complement], i]
        seen[n] = i
    return []

Group Anagrams:

from collections import defaultdict

def group_anagrams(strs):
    groups = defaultdict(list)
    for s in strs:
        key = tuple(sorted(s))   # canonical form
        groups[key].append(s)
    return list(groups.values())

Longest Consecutive Sequence (O(n)):

def longest_consecutive(nums):
    num_set = set(nums)
    best = 0
    for n in num_set:
        if n - 1 not in num_set:   # only start from sequence head
            length = 1
            while n + length in num_set:
                length += 1
            best = max(best, length)
    return best

Complexity: O(n) time, O(n) space for all three.

Pattern 2: Two Pointers

When: Sorted array, palindrome check, pair-sum problems, Dutch flag partition, removing duplicates.

Valid Palindrome:

def is_palindrome(s):
    s = ''.join(c.lower() for c in s if c.isalnum())
    l, r = 0, len(s) - 1
    while l < r:
        if s[l] != s[r]:
            return False
        l += 1; r -= 1
    return True

Container With Most Water:

def max_area(height):
    l, r = 0, len(height) - 1
    res = 0
    while l < r:
        res = max(res, min(height[l], height[r]) * (r - l))
        if height[l] < height[r]:
            l += 1
        else:
            r -= 1
    return res

3Sum (sort + two pointers):

def three_sum(nums):
    nums.sort()
    result = []
    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i-1]:
            continue                   # skip duplicate
        l, r = i + 1, len(nums) - 1
        while l < r:
            s = nums[i] + nums[l] + nums[r]
            if s == 0:
                result.append([nums[i], nums[l], nums[r]])
                while l < r and nums[l] == nums[l+1]: l += 1
                while l < r and nums[r] == nums[r-1]: r -= 1
                l += 1; r -= 1
            elif s < 0:
                l += 1
            else:
                r -= 1
    return result

Complexity: O(n) two-pointer; O(n²) for 3Sum (sort + scan).

Pattern 3: Sliding Window

When: Contiguous subarray/substring with condition (max sum, longest without repeat, count chars, minimum window).

Template (variable window):

def sliding_window(s):
    left = 0
    window = {}       # or Counter()
    res = 0
    for right in range(len(s)):
        # ---- EXPAND ----
        c = s[right]
        window[c] = window.get(c, 0) + 1

        # ---- SHRINK condition ----
        while <invalid>:
            out = s[left]
            window[out] -= 1
            if window[out] == 0:
                del window[out]
            left += 1

        # ---- RECORD ----
        res = max(res, right - left + 1)
    return res

Longest Substring Without Repeating Characters:

def length_of_longest_substring(s):
    seen = {}
    left = res = 0
    for right, c in enumerate(s):
        if c in seen and seen[c] >= left:
            left = seen[c] + 1
        seen[c] = right
        res = max(res, right - left + 1)
    return res

Minimum Window Substring:

from collections import Counter

def min_window(s, t):
    need = Counter(t)
    missing = len(t)
    best = ""
    left = 0
    for right, c in enumerate(s):
        if need[c] > 0:
            missing -= 1
        need[c] -= 1
        if missing == 0:
            # shrink from left
            while need[s[left]] < 0:
                need[s[left]] += 1
                left += 1
            window = s[left:right+1]
            if not best or len(window) < len(best):
                best = window
            # move left one step to resume search
            need[s[left]] += 1
            missing += 1
            left += 1
    return best

Complexity: O(n) time, O(k) space (k = distinct chars in window).

Pattern 4: Stack / Monotonic Stack

When: Matching brackets, expression evaluation, "next greater/smaller element", histogram areas.

Valid Parentheses:

def is_valid(s):
    stack = []
    pairs = {')': '(', '}': '{', ']': '['}
    for c in s:
        if c in '({[':
            stack.append(c)
        elif not stack or stack[-1] != pairs[c]:
            return False
        else:
            stack.pop()
    return not stack

Daily Temperatures (next greater to the right):

def daily_temperatures(temperatures):
    stack = []   # indices of pending temperatures
    result = [0] * len(temperatures)
    for i, t in enumerate(temperatures):
        while stack and temperatures[stack[-1]] < t:
            j = stack.pop()
            result[j] = i - j
        stack.append(i)
    return result

Largest Rectangle in Histogram:

def largest_rectangle_area(heights):
    stack = []    # monotonically increasing stack of indices
    max_area = 0
    heights = heights + [0]   # sentinel
    for i, h in enumerate(heights):
        start = i
        while stack and heights[stack[-1]] >= h:
            j = stack.pop()
            width = i - (stack[-1] if stack else -1) - 1
            max_area = max(max_area, heights[j] * width)
            start = j
        stack.append(i)
    return max_area

Complexity: O(n) time, O(n) stack space for all.

Pattern 5: Queues and Deques

When: BFS, sliding window maximum, task scheduling, round-robin problems.

Sliding Window Maximum (deque = O(n)):

from collections import deque

def max_sliding_window(nums, k):
    dq = deque()   # stores indices; front always holds max for current window
    result = []
    for i, n in enumerate(nums):
        # remove elements outside window
        while dq and dq[0] < i - k + 1:
            dq.popleft()
        # remove smaller elements from the back
        while dq and nums[dq[-1]] < n:
            dq.pop()
        dq.append(i)
        if i >= k - 1:
            result.append(nums[dq[0]])
    return result

Complexity: O(n) time, O(k) space.

Pattern 6: Linked Lists

When: Explicitly given a linked list; in-place reversal; cycle detection; merge operations.

Node definition + Reverse:

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def reverse_list(head):
    prev = None
    curr = head
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt
    return prev

Floyd's Cycle Detection:

def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            return True
    return False

Merge Two Sorted Lists:

def merge_two_lists(l1, l2):
    dummy = ListNode()
    cur = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            cur.next = l1; l1 = l1.next
        else:
            cur.next = l2; l2 = l2.next
        cur = cur.next
    cur.next = l1 or l2
    return dummy.next

Pattern 7: Trees — BFS and DFS

When: Any tree/binary tree problem. BFS for level-by-level; DFS for path problems, subtree values, recursive structure.

TreeNode definition:

class TreeNode:
    def __init__(self, val=0, left=None, right=None):

        self.left = left
        self.right = right

Recursive DFS — Max Depth:

def max_depth(root):
    if not root:
        return 0
    return 1 + max(max_depth(root.left), max_depth(root.right))

Iterative DFS (preorder with stack):

def preorder(root):
    if not root: return []
    stack, result = [root], []
    while stack:
        node = stack.pop()
        result.append(node.val)
        if node.right: stack.append(node.right)   # right first so left processed first
        if node.left:  stack.append(node.left)
    return result

BFS (level order):

from collections import deque

def level_order(root):
    if not root: return []
    queue = deque([root])
    result = []
    while queue:
        level = []
        for _ in range(len(queue)):
            node = queue.popleft()
            level.append(node.val)
            if node.left:  queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)
    return result

Validate BST (in-order + prev tracking):

def is_valid_bst(root):
    prev = [float('-inf')]
    def inorder(node):
        if not node: return True
        if not inorder(node.left): return False
        if node.val <= prev[0]: return False
        prev[0] = node.val
        return inorder(node.right)
    return inorder(root)

Lowest Common Ancestor (BST):

def lca(root, p, q):
    while root:
        if p.val < root.val and q.val < root.val:
            root = root.left
        elif p.val > root.val and q.val > root.val:
            root = root.right
        else:
            return root
    return None

Complexity: O(n) time, O(h) space (h = height; worst O(n) skewed, O(log n) balanced).

Pattern 8: Graphs — BFS / DFS / Union-Find / Topological Sort

When: Connectivity, shortest path (unweighted), cycle detection, scheduling, islands.

Adjacency list + BFS:

from collections import defaultdict, deque

def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    order = []
    while queue:
        node = queue.popleft()
        order.append(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return order

DFS iterative:

def dfs(graph, start):
    visited = set()
    stack = [start]
    while stack:
        node = stack.pop()
        if node not in visited:
            visited.add(node)
            stack.extend(graph[node])

Number of Islands (grid DFS):

def num_islands(grid):
    if not grid: return 0
    rows, cols = len(grid), len(grid[0])
    count = 0

    def dfs(r, c):
        if r < 0 or r >= rows or c < 0 or c >= cols or grid[r][c] != '1':
            return
        grid[r][c] = '0'    # mark visited in-place
        for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
            dfs(r+dr, c+dc)

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                dfs(r, c)
                count += 1
    return count

Topological Sort (Kahn's BFS — for Course Schedule):

from collections import deque, defaultdict

def can_finish(num_courses, prerequisites):
    graph = defaultdict(list)
    in_degree = [0] * num_courses
    for a, b in prerequisites:
        graph[b].append(a)
        in_degree[a] += 1
    queue = deque(i for i in range(num_courses) if in_degree[i] == 0)
    completed = 0
    while queue:
        node = queue.popleft()
        completed += 1
        for neighbor in graph[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    return completed == num_courses

Union-Find (with path compression + union by rank):

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]   # path compression (halving)
            x = self.parent[x]
        return x

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb: return False
        if self.rank[ra] < self.rank[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        if self.rank[ra] == self.rank[rb]:
            self.rank[ra] += 1
        return True

Complexity: BFS/DFS O(V+E); Union-Find O(α(n)) amortized ≈ O(1) in practice.

Pattern 9: Heaps / Priority Queue

When: k-th largest/smallest, top-K elements, merge K sorted lists, streaming median, task scheduling.

Python heapq is a min-heap. For max-heap, negate values.

Kth Largest Element:

import heapq

def find_kth_largest(nums, k):
    # maintain a min-heap of size k
    heap = []
    for n in nums:
        heapq.heappush(heap, n)
        if len(heap) > k:
            heapq.heappop(heap)
    return heap[0]   # k-th largest = min of top-k heap

Top K Frequent Elements:

from collections import Counter
import heapq

def top_k_frequent(nums, k):
    freq = Counter(nums)
    # heap of (freq, num); min-heap → pop least frequent
    heap = []
    for num, count in freq.items():
        heapq.heappush(heap, (count, num))
        if len(heap) > k:
            heapq.heappop(heap)
    return [num for count, num in heap]

Merge K Sorted Lists:

import heapq

def merge_k_lists(lists):
    heap = []
    dummy = ListNode()
    cur = dummy
    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))
    while heap:
        val, i, node = heapq.heappop(heap)
        cur.next = node
        cur = cur.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))

Pattern 10: Binary Search

When: Sorted array, monotonic condition ("find minimum valid X"), "find boundary" problems.

Classic binary search:

def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1

Find leftmost position (lower bound):

def lower_bound(arr, target):

        else:

Search in Rotated Sorted Array:

def search_rotated(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if nums[mid] == target:
            return mid
        # left half sorted
        if nums[lo] <= nums[mid]:
            if nums[lo] <= target < nums[mid]:
                hi = mid - 1
            else:
                lo = mid + 1
        # right half sorted
        else:
            if nums[mid] < target <= nums[hi]:

            else:

    return -1

Binary search on answer (Capacity to Ship in D Days):

def ship_within_days(weights, days):

Pattern 11: Recursion / Backtracking

When: Exhaustive search, permutations, combinations, constraint satisfaction (Sudoku, N-Queens, word search).

Template:

def backtrack(start, path, result, candidates, target):
    if <base case / goal reached>:
        result.append(path[:])   # copy!
        return
    for i in range(start, len(candidates)):
        if <pruning condition>:
            continue
        path.append(candidates[i])
        backtrack(i + 1, path, result, candidates, target)  # i+1 = no reuse; i = reuse
        path.pop()                                            # undo choice

Subsets:

def subsets(nums):
    result = []
    def backtrack(start, path):
        result.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()
    backtrack(0, [])
    return result

Combination Sum (reuse allowed):

def combination_sum(candidates, target):
    result = []
    candidates.sort()
    def backtrack(start, path, remaining):
        if remaining == 0:
            result.append(path[:])
            return
        for i in range(start, len(candidates)):
            if candidates[i] > remaining:
                break   # pruning
            path.append(candidates[i])
            backtrack(i, path, remaining - candidates[i])   # i not i+1 = reuse
            path.pop()
    backtrack(0, [], target)
    return result

Word Search (grid backtracking):

def exist(board, word):
    rows, cols = len(board), len(board[0])

    def dfs(r, c, idx):
        if idx == len(word): return True
        if r < 0 or r >= rows or c < 0 or c >= cols: return False
        if board[r][c] != word[idx]: return False
        tmp = board[r][c]
        board[r][c] = '#'   # mark visited
        found = any(dfs(r+dr, c+dc, idx+1) for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)])
        board[r][c] = tmp   # restore
        return found

    return any(dfs(r, c, 0) for r in range(rows) for c in range(cols))

Complexity: O(b^d) worst case (b = branching factor, d = depth). Pruning is what makes it practical.

Pattern 12: Dynamic Programming

When: Overlapping subproblems + optimal substructure; max/min/count across choices.

Approaches: - Top-down: recursive + @functools.lru_cache(maxsize=None) — write naturally, memoize automatically - Bottom-up: explicit table — better control over space optimization (rolling array)

Coin Change (1D DP — minimum coins):

def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for a in range(1, amount + 1):
        for c in coins:
            if c <= a:
                dp[a] = min(dp[a], dp[a - c] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

Longest Common Subsequence (2D DP):

def lcs(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i-1] == text2[j-1]:
                dp[i][j] = 1 + dp[i-1][j-1]
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]

Edit Distance:

def min_distance(word1, word2):
    m, n = len(word1), len(word2)
    dp = [[0]*(n+1) for _ in range(m+1)]
    for i in range(m+1): dp[i][0] = i
    for j in range(n+1): dp[0][j] = j
    for i in range(1, m+1):
        for j in range(1, n+1):
            if word1[i-1] == word2[j-1]:
                dp[i][j] = dp[i-1][j-1]
            else:
                dp[i][j] = 1 + min(dp[i-1][j],    # delete
                                    dp[i][j-1],    # insert
                                    dp[i-1][j-1])  # replace
    return dp[m][n]

0/1 Knapsack:

def knapsack(weights, values, capacity):
    n = len(weights)
    dp = [0] * (capacity + 1)
    for i in range(n):
        for w in range(capacity, weights[i] - 1, -1):   # reverse to avoid reuse
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[capacity]

Memoized top-down with lru_cache:

import functools

def longest_increasing_subsequence(nums):

3A. k-Means Clustering

import math
import random


def euclidean(a, b):
    return math.sqrt(sum((x - y) ** 2 for x, y in zip(a, b)))


def k_means(data, k, max_iter=100, seed=42):
    """
    Lloyd's algorithm for k-means.
    - data: list of d-dimensional points (list of lists)
    - k: number of clusters
    Returns: (labels, centroids)

    Key facts to state:
    - Time complexity: O(n * k * d * max_iter)
    - Assumes spherical, similarly-sized clusters
    - Sensitive to scale: normalize features first
    - k-means++ init reduces bad convergence probability
    - Inertia (WCSS) = sum of squared distances to nearest centroid
    """
    random.seed(seed)
    centroids = random.sample(data, k)

    for _ in range(max_iter):
        # ASSIGN each point to nearest centroid
        clusters = [[] for _ in range(k)]
        for point in data:
            dists = [euclidean(point, c) for c in centroids]
            clusters[dists.index(min(dists))].append(point)

        # UPDATE centroids to cluster means
        new_centroids = []
        for i, cluster in enumerate(clusters):
            if cluster:
                d = len(cluster[0])
                new_c = [sum(p[dim] for p in cluster) / len(cluster) for dim in range(d)]
                new_centroids.append(new_c)
            else:
                new_centroids.append(random.choice(data))   # empty cluster: reinit

        # CONVERGENCE check
        if all(euclidean(centroids[i], new_centroids[i]) < 1e-6 for i in range(k)):
            break
        centroids = new_centroids

    labels = [min(range(k), key=lambda i: euclidean(p, centroids[i])) for p in data]
    return labels, centroids

import numpy as np
from collections import Counter


class KNN:
    """
    k-Nearest Neighbors for classification (and optionally regression).

    Key facts:
    - Lazy learner: no training, O(n*d) per prediction
    - Small k = high variance (overfits); large k = high bias (underfits)
    - Sensitive to feature scale: normalize before using
    - Curse of dimensionality: distance concentrates in high d
    - For regression: return np.mean(k_labels) instead of majority vote
    """

    def __init__(self, k=3):
        self.k = k
        self.X_train = None
        self.y_train = None

    def fit(self, X, y):
        self.X_train = np.array(X, dtype=float)
        self.y_train = np.array(y)

    def _distance(self, a, b):
        return np.sqrt(np.sum((a - b) ** 2))

    def predict(self, X):
        return np.array([self._predict_one(x) for x in np.array(X, dtype=float)])

    def _predict_one(self, x):
        dists = [self._distance(x, xt) for xt in self.X_train]
        k_indices = np.argsort(dists)[:self.k]
        k_labels = self.y_train[k_indices]
        return Counter(k_labels).most_common(1)[0][0]   # classification


# numpy-vectorized version (much faster; use this if numpy available)
class KNNVectorized:
    def __init__(self, k=3):
        self.k = k

    def fit(self, X, y):
        self.X_train = np.array(X, dtype=float)
        self.y_train = np.array(y)

    def predict(self, X):

        # Euclidean distance via broadcasting: (n_test, n_train)
        diff = X[:, np.newaxis, :] - self.X_train[np.newaxis, :, :]
        dists = np.sqrt((diff ** 2).sum(axis=-1))
        k_idx = np.argsort(dists, axis=1)[:, :self.k]
        k_labels = self.y_train[k_idx]
        # majority vote
        return np.array([Counter(row).most_common(1)[0][0] for row in k_labels])

3C. Linear Regression with Gradient Descent

import numpy as np


class LinearRegression:
    """
    Ordinary least squares via gradient descent.

    Loss: MSE = (1/n) * ||Xw + b - y||^2
    Gradient w.r.t. w: (2/n) * X^T (Xw + b - y)
    Gradient w.r.t. b: (2/n) * sum(Xw + b - y)

    Analytical solution (Normal Equations): w = (X^T X)^{-1} X^T y
      - Use when n is manageable (< ~10k samples)
      - O(n*d^2 + d^3); gradient descent O(n*d) per iteration
    """

    def __init__(self, lr=0.01, n_iter=1000):
        self.lr = lr
        self.n_iter = n_iter
        self.w = None
        self.b = None

    def fit(self, X, y):
        X, y = np.array(X, dtype=float), np.array(y, dtype=float)
        n, d = X.shape
        self.w = np.zeros(d)
        self.b = 0.0
        for _ in range(self.n_iter):
            y_pred = X @ self.w + self.b
            error = y_pred - y
            dw = (2 / n) * X.T @ error
            db = (2 / n) * np.sum(error)
            self.w -= self.lr * dw
            self.b  -= self.lr * db

    def predict(self, X):
        return np.array(X, dtype=float) @ self.w + self.b

    @staticmethod
    def normal_equations(X, y):
        """Analytical solution via normal equations."""
        X = np.array(X, dtype=float)
        y = np.array(y, dtype=float)
        # Add bias column
        X_b = np.column_stack([np.ones(len(X)), X])
        # w = (X^T X)^{-1} X^T y
        return np.linalg.lstsq(X_b, y, rcond=None)[0]

3D. Logistic Regression with Gradient Descent

import numpy as np


class LogisticRegression:
    """
    Binary logistic regression via gradient descent.

    Model: p = sigmoid(Xw + b)
    Loss: Binary cross-entropy = -(1/n) * sum(y*log(p) + (1-y)*log(1-p))
    Gradient w.r.t. w: (1/n) * X^T (p - y)     <- same form as linear!
    Gradient w.r.t. b: (1/n) * sum(p - y)

    L2 regularization: add (lambda/n)*w to dw (do NOT regularize bias)
    Multi-class: use softmax + cross-entropy (see Section 3F)
    """

    def __init__(self, lr=0.01, n_iter=1000, lambda_=0.0):
        self.lr = lr
        self.n_iter = n_iter
        self.lambda_ = lambda_
        self.w = None
        self.b = None

    @staticmethod
    def _sigmoid(z):
        return 1.0 / (1.0 + np.exp(-np.clip(z, -500, 500)))   # clip: numerical stability

    def fit(self, X, y):
        X, y = np.array(X, dtype=float), np.array(y, dtype=float)
        n, d = X.shape
        self.w = np.zeros(d)
        self.b = 0.0
        for _ in range(self.n_iter):
            z = X @ self.w + self.b
            p = self._sigmoid(z)
            dw = (1 / n) * X.T @ (p - y) + (self.lambda_ / n) * self.w
            db = (1 / n) * np.sum(p - y)
            self.w -= self.lr * dw
            self.b  -= self.lr * db

    def predict_proba(self, X):
        return self._sigmoid(np.array(X, dtype=float) @ self.w + self.b)

    def predict(self, X, threshold=0.5):
        return (self.predict_proba(X) >= threshold).astype(int)

    def log_loss(self, X, y):
        """Compute binary cross-entropy loss on (X, y)."""
        p = np.clip(self.predict_proba(X), 1e-12, 1 - 1e-12)
        y = np.array(y)
        return -np.mean(y * np.log(p) + (1 - y) * np.log(1 - p))

3E. Train/Test Split and Evaluation Metrics

import numpy as np


def train_test_split(X, y, test_size=0.2, seed=42):
    """Shuffle and split into train/test."""
    X, y = np.array(X), np.array(y)
    np.random.seed(seed)
    idx = np.random.permutation(len(X))
    split = int(len(X) * (1 - test_size))
    train_idx, test_idx = idx[:split], idx[split:]
    return X[train_idx], X[test_idx], y[train_idx], y[test_idx]


def accuracy(y_true, y_pred):
    return float(np.mean(np.array(y_true) == np.array(y_pred)))


def confusion_matrix_counts(y_true, y_pred):
    y_true, y_pred = np.array(y_true), np.array(y_pred)
    TP = int(np.sum((y_pred == 1) & (y_true == 1)))
    FP = int(np.sum((y_pred == 1) & (y_true == 0)))
    FN = int(np.sum((y_pred == 0) & (y_true == 1)))
    TN = int(np.sum((y_pred == 0) & (y_true == 0)))
    return TP, FP, FN, TN


def precision_recall_f1(y_true, y_pred):
    """
    Precision = TP / (TP + FP)  <- "of what we predicted positive, how many were right?"
    Recall    = TP / (TP + FN)  <- "of all actual positives, how many did we find?"
    F1        = harmonic mean of P and R  <- balanced metric; prefer over accuracy when imbalanced
    """
    TP, FP, FN, TN = confusion_matrix_counts(y_true, y_pred)
    p  = TP / (TP + FP) if (TP + FP) > 0 else 0.0
    r  = TP / (TP + FN) if (TP + FN) > 0 else 0.0
    f1 = 2 * p * r / (p + r) if (p + r) > 0 else 0.0
    return p, r, f1


def roc_auc(y_true, y_scores):
    """
    AUC-ROC = probability that a random positive is ranked above a random negative.
    AUC = 0.5: random; AUC = 1.0: perfect; AUC < 0.5: worse than random.
    Threshold-independent — use when class imbalance exists.
    """
    y_true = np.array(y_true)
    y_scores = np.array(y_scores)
    order = np.argsort(-y_scores)
    y_sorted = y_true[order]
    P = np.sum(y_true)
    N = len(y_true) - P
    if P == 0 or N == 0:
        return 0.0
    tps, fps = 0, 0
    tpr_list = [0.0]
    fpr_list = [0.0]
    for label in y_sorted:
        if label == 1:
            tps += 1
        else:
            fps += 1
        tpr_list.append(tps / P)
        fpr_list.append(fps / N)
    # Trapezoidal rule
    auc = float(np.trapz(tpr_list, fpr_list))
    return abs(auc)


# --- k-fold cross validation scaffold ---
def k_fold_cross_val(model_class, X, y, k=5, model_kwargs=None):
    """Returns list of fold accuracies."""
    model_kwargs = model_kwargs or {}
    X, y = np.array(X), np.array(y)
    n = len(X)
    indices = np.random.permutation(n)
    fold_size = n // k
    scores = []
    for fold in range(k):
        val_idx = indices[fold * fold_size:(fold + 1) * fold_size]
        train_idx = np.concatenate([indices[:fold * fold_size], indices[(fold + 1) * fold_size:]])
        model = model_class(**model_kwargs)
        model.fit(X[train_idx], y[train_idx])
        preds = model.predict(X[val_idx])
        scores.append(accuracy(y[val_idx], preds))
    return scores

3F. Softmax and Cross-Entropy Loss

import numpy as np


def softmax(logits):
    """
    Numerically stable softmax via LogSumExp trick: subtract max before exp.
    Without this: exp(1000) = inf in float32/64.

    Formula: softmax(z)_i = exp(z_i) / sum_j exp(z_j)
    Shaped for batches: logits shape (n, C)
    """
    shifted = logits - np.max(logits, axis=-1, keepdims=True)
    exp_vals = np.exp(shifted)
    return exp_vals / np.sum(exp_vals, axis=-1, keepdims=True)


def cross_entropy_loss(logits, labels):
    """
    labels: integer class indices, shape (n,)
    Returns scalar mean loss.

    Loss = -(1/n) * sum_i log( softmax(logits_i)[labels_i] )
    Equivalently: NLL of the categorical distribution.
    """
    probs = softmax(logits)
    n = logits.shape[0]
    # index into each row to get the probability of the true class
    true_probs = probs[np.arange(n), labels]
    log_probs = -np.log(np.clip(true_probs, 1e-12, 1.0))
    return float(np.mean(log_probs))


def softmax_cross_entropy_gradient(logits, labels):
    """
    d(CE) / d(logits) = (softmax(logits) - one_hot(labels)) / n
    This is the elegant combined gradient; no need to separate softmax gradient.
    """
    probs = softmax(logits)
    n = logits.shape[0]
    probs[np.arange(n), labels] -= 1
    return probs / n


# One-hot encoding helper
def one_hot(labels, num_classes):
    n = len(labels)
    oh = np.zeros((n, num_classes))
    oh[np.arange(n), labels] = 1
    return oh

3G. Tiny MLP — Forward and Backward Pass

import numpy as np


def relu(x):
    return np.maximum(0, x)


def relu_grad(x):
    return (x > 0).astype(float)


class TinyMLP:
    """
    2-layer MLP: input -> hidden (ReLU) -> output (softmax + CE loss).

    Architecture:
        z1 = X @ W1 + b1        (n, hidden_dim)
        a1 = relu(z1)            (n, hidden_dim)
        z2 = a1 @ W2 + b2       (n, num_classes)
        loss = CE(z2, labels)

    Backprop:
        dL/dz2 = (softmax(z2) - one_hot) / n    [gradient of CE + softmax combined]
        dL/dW2 = a1.T @ dz2
        dL/db2 = sum(dz2, axis=0)
        dL/da1 = dz2 @ W2.T
        dL/dz1 = da1 * relu_grad(z1)            [chain rule through ReLU]
        dL/dW1 = X.T @ dz1
        dL/db1 = sum(dz1, axis=0)
    """

    def __init__(self, input_dim, hidden_dim, num_classes, lr=0.01):
        # Xavier initialization
        self.W1 = np.random.randn(input_dim, hidden_dim) * np.sqrt(2.0 / input_dim)
        self.b1 = np.zeros(hidden_dim)
        self.W2 = np.random.randn(hidden_dim, num_classes) * np.sqrt(2.0 / hidden_dim)
        self.b2 = np.zeros(num_classes)
        self.lr = lr

    def forward(self, X):
        """Returns (logits, cache) where cache has intermediate values for backward."""
        z1 = X @ self.W1 + self.b1
        a1 = relu(z1)
        z2 = a1 @ self.W2 + self.b2
        cache = (X, z1, a1, z2)
        return z2, cache

    def backward(self, cache, labels):
        X, z1, a1, z2 = cache
        n = X.shape[0]

        # Gradient of CE + softmax combined
        probs = softmax(z2)
        dz2 = probs.copy()
        dz2[np.arange(n), labels] -= 1
        dz2 /= n

        dW2 = a1.T @ dz2
        db2 = dz2.sum(axis=0)

        da1 = dz2 @ self.W2.T
        dz1 = da1 * relu_grad(z1)    # chain rule: ReLU gate

        dW1 = X.T @ dz1
        db1 = dz1.sum(axis=0)

        # Gradient descent update
        self.W2 -= self.lr * dW2
        self.b2 -= self.lr * db2
        self.W1 -= self.lr * dW1
        self.b1 -= self.lr * db1

    def train_step(self, X, labels):
        logits, cache = self.forward(X)
        loss = cross_entropy_loss(logits, labels)
        self.backward(cache, labels)
        return loss

    def predict(self, X):
        logits, _ = self.forward(X)
        return np.argmax(logits, axis=1)


def softmax(logits):
    shifted = logits - np.max(logits, axis=-1, keepdims=True)
    e = np.exp(shifted)
    return e / e.sum(axis=-1, keepdims=True)


def cross_entropy_loss(logits, labels):
    probs = softmax(logits)
    n = logits.shape[0]
    return float(-np.mean(np.log(np.clip(probs[np.arange(n), labels], 1e-12, 1.0))))

3H. Scaled Dot-Product Attention

import numpy as np


def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Core attention operation (Vaswani et al. 2017).

    Formula: Attention(Q, K, V) = softmax(Q K^T / sqrt(d_k)) * V

    Args:
        Q: (..., seq_q, d_k)
        K: (..., seq_k, d_k)
        V: (..., seq_k, d_v)
        mask: bool/int mask; 0 positions are masked (set to -inf before softmax)

    Returns: output (..., seq_q, d_v), attn_weights (..., seq_q, seq_k)

    Key facts:
    - sqrt(d_k) scaling: dot products grow O(d_k) in expectation, pushing softmax
      into saturation (near-zero gradients). Dividing by sqrt(d_k) normalizes variance.
    - Causal mask (decoder self-attention): mask upper triangle -> -1e9 before softmax
      so each position only attends to past positions.
    - Multi-head: project Q/K/V into h sub-spaces, run attention in parallel,
      concatenate outputs: MultiHead(Q,K,V) = Concat(head_1, ..., head_h) W_O
    - Complexity: O(n^2 * d_k) per head -> quadratic in sequence length.
      Motivation for FlashAttention (IO-aware tiling), linear attention, GQA, MLA.
    - GQA (LLaMA 3, Mistral): G groups share K/V heads -> reduce KV cache G-fold.
    """
    d_k = Q.shape[-1]
    # (..., seq_q, seq_k)
    scores = Q @ K.swapaxes(-2, -1) / np.sqrt(d_k)

    if mask is not None:
        # Where mask == 0, set to large negative (zeroed after softmax)
        scores = np.where(mask == 0, -1e9, scores)

    # Numerically stable softmax over key dimension
    scores = scores - np.max(scores, axis=-1, keepdims=True)
    exp_scores = np.exp(scores)
    attn_weights = exp_scores / (np.sum(exp_scores, axis=-1, keepdims=True) + 1e-9)

    output = attn_weights @ V
    return output, attn_weights


def make_causal_mask(seq_len):
    """Lower-triangular mask: position i can only attend to j <= i."""
    return np.tril(np.ones((seq_len, seq_len), dtype=int))


def multi_head_attention(Q, K, V, W_Q, W_K, W_V, W_O, n_heads, mask=None):
    """
    Minimal multi-head attention.
    Q/K/V: (batch, seq, d_model)
    W_Q/W_K/W_V: (d_model, d_model)  -- projects then splits into heads
    W_O: (d_model, d_model)
    """
    batch, seq, d_model = Q.shape
    d_head = d_model // n_heads

    def split_heads(x):
        x = x @ (W_Q if x is Q else (W_K if x is K else W_V))
        x = x.reshape(batch, seq, n_heads, d_head)
        return x.transpose(0, 2, 1, 3)   # (batch, heads, seq, d_head)

    Q_ = (Q @ W_Q).reshape(batch, seq, n_heads, d_head).transpose(0, 2, 1, 3)
    K_ = (K @ W_K).reshape(batch, seq, n_heads, d_head).transpose(0, 2, 1, 3)
    V_ = (V @ W_V).reshape(batch, seq, n_heads, d_head).transpose(0, 2, 1, 3)

    out, weights = scaled_dot_product_attention(Q_, K_, V_, mask=mask)
    # (batch, heads, seq, d_head) -> (batch, seq, d_model)
    out = out.transpose(0, 2, 1, 3).reshape(batch, seq, d_model)
    return out @ W_O, weights

3I. Minimal BPE Tokenizer

from collections import Counter
from collections import deque


class BPETokenizer:
    """
    Byte-Pair Encoding tokenizer (Sennrich et al. 2016 style, byte-level like GPT-2).

    Algorithm:
    1. Start with all 256 UTF-8 byte IDs as initial vocabulary.
    2. Count all adjacent pairs in the current token sequence.
    3. Merge the most frequent pair -> assign it a new ID.
    4. Repeat until vocab_size is reached.

    Key facts:
    - Merge ORDER is critical; encoding must replay merges in the same sequence.
    - Byte-level BPE = zero OOV tokens (any text is valid UTF-8 bytes).
    - Larger vocab_size: shorter sequences (cheaper attention) but larger embedding matrices.
    - eBay's LiLiuM uses a custom vocabulary tuned for e-commerce terms -> 34% speedup.
    - Used in GPT-2/3/4, LLaMA 3 (128k vocab), Qwen 3.
    - Alternative: SentencePiece Unigram (used in Gemma, mT5) - probabilistic, no whitespace assumption.
    """

    def __init__(self, vocab_size=500):
        self.vocab_size = vocab_size
        self.merges = {}    # (id1, id2) -> new_id; ordered dict preserves merge order
        self.vocab = {}     # id -> bytes

    def _get_pair_freqs(self, ids):
        return Counter(zip(ids, ids[1:]))

    def _merge_pair(self, ids, pair, new_id):
        dq = deque(ids)
        result = []
        while dq:
            cur = dq.popleft()
            if dq and (cur, dq[0]) == pair:
                result.append(new_id)
                dq.popleft()        # consume the second element of the pair
            else:
                result.append(cur)
        return result

    def train(self, text):
        """Train BPE on raw text string."""
        ids = list(text.encode('utf-8'))
        # Init vocab with all 256 bytes
        self.vocab = {i: bytes([i]) for i in range(256)}
        next_id = 256

        while next_id < self.vocab_size:
            pair_freqs = self._get_pair_freqs(ids)
            if not pair_freqs:
                break
            best_pair = max(pair_freqs, key=pair_freqs.get)
            self.merges[best_pair] = next_id
            self.vocab[next_id] = self.vocab[best_pair[0]] + self.vocab[best_pair[1]]
            ids = self._merge_pair(ids, best_pair, next_id)
            next_id += 1

    def encode(self, text):
        """Encode text to list of token IDs by replaying merges in order."""
        ids = list(text.encode('utf-8'))
        for pair, new_id in self.merges.items():
            ids = self._merge_pair(ids, pair, new_id)
        return ids

    def decode(self, ids):
        """Decode list of token IDs back to string."""
        byte_seq = b''.join(self.vocab[i] for i in ids)
        return byte_seq.decode('utf-8', errors='replace')

    def token_count(self, text):
        return len(self.encode(text))

4A. Bayes' Theorem

Formula: P(A|B) = P(B|A) * P(A) / P(B)

where P(B) = P(B|A)P(A) + P(B|not A)P(not A)

Canonical diagnostic test problem:

A disease has prevalence 1%. A test has 99% sensitivity (TP rate) and 99% specificity (TN rate). If someone tests positive, what is P(they have the disease)?

P(D) = 0.01,   P(+|D) = 0.99,   P(+|no D) = 1 - specificity = 0.01

P(D|+) = P(+|D) * P(D) / [P(+|D)*P(D) + P(+|no D)*P(no D)]
       = (0.99 * 0.01) / (0.99*0.01 + 0.01*0.99)
       = 0.0099 / 0.0198
       = 0.50 = 50%

Counterintuitive takeaway: Even a 99%-accurate test produces 50% false positive rate when disease is rare (prevalence = 1%). The prior probability dominates when it is extreme. This is the base rate fallacy — a classic MCQ trap.

Coin variant (FAANG standard):

Pick uniformly between a fair coin and a two-heads coin. Flip 5 times, all heads. P(two-heads coin)?

P(H5|fair) = (1/2)^5 = 1/32
P(H5|unfair) = 1
P(each) = 0.5

P(unfair|H5) = 1*0.5 / (1*0.5 + 1/32 * 0.5) = 0.5 / (0.5 + 0.015625) ≈ 0.970

4B. Expected Value and Recursive Conditioning

Expected coin flips until HH (two consecutive heads):

Let E = expected number of flips. Condition on first two outcomes:

Event Prob Outcome
T (first flip tails) 1/2 Must restart after 1 flip: 1 + E
HT (heads then tails) 1/4 Must restart after 2 flips: 2 + E
HH (success) 1/4 Done in 2 flips: 2 
E = (1/2)(1 + E) + (1/4)(2 + E) + (1/4)(2)
E = 1/2 + E/2 + 1/2 + E/4 + 1/2
E - E/2 - E/4 = 3/2
E/4 = 3/2
E = 6

Expected rolls of a die to get all 6 faces (Coupon Collector):

When k faces already seen, expected rolls to see a new face = 6/(6-k).

E[total] = 6/6 + 6/5 + 6/4 + 6/3 + 6/2 + 6/1
         = 1 + 1.2 + 1.5 + 2 + 3 + 6 = 14.7

Formula: E[T] = n * H_n where H_n is the n-th harmonic number.

Expected value from game with restart:

You roll a die. If you roll 6, you win $10. Otherwise pay $1 and roll again. What's the expected cost?

Let E = expected cost (not counting final win).

P(6) = 1/6: pay $0 in cost, win $10 (treat cost as 0)
P(not 6) = 5/6: pay $1, then face same problem: E[cost] = 1 + E

E = (5/6)(1 + E) + (1/6)(0)
E = 5/6 + 5E/6
E/6 = 5/6
E = 5    (expected cost = $5 before winning)

4C. Combinatorics and Counting

Chords on a circle:

4 points on a circle, randomly partitioned into 2 chords. P(chords intersect)?

Label points A, B, C, D in order around circle. Three ways to pair them: - (AB, CD) — no intersection - (AC, BD) — intersect - (AD, BC) — no intersection

4D. Hypothesis Testing and CLT

Hypothesis test on coin flips:

You flip a coin 1000 times and get 550 heads. Is the coin fair? (alpha = 0.05)

H0: p = 0.5   H1: p != 0.5

Under H0:
  E[X] = 500
  std  = sqrt(1000 * 0.5 * 0.5) = 15.81

z = (550 - 500) / 15.81 = 3.16

P(|Z| > 3.16) = 2 * P(Z > 3.16) ≈ 2 * 0.0008 = 0.0016 < 0.05

Conclusion: reject H0 at 5% significance. Coin is likely biased.

Key CLT facts for MCQs: - For n independent X_i with mean μ and variance σ²: (X̄ - μ) / (σ/√n) → N(0,1) as n→∞ - Rule of thumb: CLT applies when n ≥ 30 - Standard error of mean = σ/√n (gets smaller as n grows) - 95% confidence interval: X̄ ± 1.96 * σ/√n

4F. A/B Testing Basics (common in ML MCQ)

Setup: Control group (A) with n_A users, conversion rate p_A. Treatment group (B) with n_B users, conversion rate p_B.

Test statistic:

z = (p_B - p_A) / sqrt(p_pool*(1-p_pool) * (1/n_A + 1/n_B))
where p_pool = (x_A + x_B) / (n_A + n_B)

Common MCQ traps: 1. "We ran the test until significance." -> peeking problem; inflates false positive rate. 2. "Test ran 3 days, got p=0.04." -> insufficient time (day-of-week effects, novelty effect). 3. "Both tests are independent." -> verify: same time period, no user overlap, no spillover. 4. "Multiple testing on 10 metrics." -> need Bonferroni correction: use α/10 per test.

5. ML Concept MCQ-Style Q&A

For Module 2 of the ML assessment. Read each question, formulate your answer mentally, then check the model answer.

Q1. What does an AUC-ROC of 0.5 mean?

A: The model performs identically to random chance — for any threshold, TPR = FPR. This means the model has learned no useful discriminative pattern. AUC < 0.5 means the model is anti-correlated with the truth (flip predictions to get AUC = 1 - original).

Q2. Training accuracy 99%, test accuracy 65%. What is wrong? How do you fix it?

A: Overfitting (high variance). The model memorizes training data but fails to generalize. Fixes: (1) more training data, (2) L1/L2 regularization, (3) dropout, (4) simpler model (fewer parameters), (5) early stopping on validation loss, (6) data augmentation.

Q3. Both training accuracy and test accuracy are 60% on a binary classification problem where baseline (majority class) is 58%. What is wrong?

A: High bias (underfitting). The model lacks capacity to learn the data patterns. Fixes: (1) increase model complexity, (2) add features, (3) reduce regularization, (4) train longer, (5) use non-linear model.

Q4. When would you use F1 score over accuracy?

A: When class distribution is imbalanced. Example: fraud detection where 99% of transactions are non-fraud. A model that always predicts non-fraud gets 99% accuracy but 0% recall for fraud. F1 = 2PR/(P+R) penalizes both missing positives (low recall) and false alarms (low precision). Use F1 when both error types matter and classes are imbalanced.

Q5. What is the difference between L1 and L2 regularization?

A: - L1 (Lasso): adds |w| penalty; optimization landscape has corners at zero → drives some weights to exactly 0 → sparse models → effective for feature selection when many features are irrelevant. - L2 (Ridge): adds w² penalty; gradient is 2λw; shrinks weights proportionally but rarely to 0; closed-form solution exists; handles multicollinearity by distributing weight across correlated features. - Elastic Net: combines both; controls sparsity and group selection simultaneously.

Q6. Explain the bias-variance tradeoff.

A: Total expected error = Bias² + Variance + Irreducible noise. - Bias: systematic error from wrong model assumptions (underfitting). Fixed by more capacity. - Variance: sensitivity to training data noise (overfitting). Fixed by regularization, more data. - Tradeoff: reducing bias usually increases variance, and vice versa. The optimal model complexity minimizes their sum on the validation set. - As model size grows: bias decreases monotonically; variance first increases then (for large overparameterized models like deep nets) can decrease again — the double-descent phenomenon.

Q7. Random Forest vs. Gradient Boosting — when to use each?

A: - Random Forest (bagging): trains N trees in parallel on bootstrap samples with random feature subsets; averages/votes. Reduces variance. Fast to train. Robust to hyperparameters. Good baseline. - Gradient Boosting (XGBoost/LightGBM): trains trees sequentially; each tree fits residuals of the previous. Reduces bias. Slower (sequential). More sensitive to hyperparameters. Usually higher accuracy with tuning. Use when you have time to tune and need best performance. - Prefer Random Forest when: fast training matters, data is noisy, you need a solid out-of-box baseline. - Prefer XGBoost/LightGBM when: accuracy is paramount, data is tabular, you will tune hyperparameters.

Q8. What happens when the learning rate is too high? Too low?

A: - Too high: loss oscillates or diverges; gradient steps overshoot the minimum. Symptom: loss spikes or goes NaN. - Too low: training converges extremely slowly; model may get stuck in a suboptimal region; wastes compute. - Standard practice: warmup (ramp up from 0 to target LR over 1k–2k steps) + cosine decay. For LLM pretraining (Versley's domain), this is the universal recipe. Adam's adaptive per-parameter rates reduce sensitivity to global LR but it still matters.

Q9. What is log-loss (cross-entropy) and why prefer it over accuracy for classification?

A: - Log-loss = -(1/n) * Σ [ylog(p) + (1-y)log(1-p)] for binary; -(1/n)Σ y_ilog(p_i) for multi-class. - Penalizes confident wrong predictions heavily (log(p) → -inf as p→0 for a true class). - Differentiable and convex → amenable to gradient-based optimization. - Measures probabilistic calibration, not just correctness. A model that says "60% confident" every time for all correct predictions has 100% accuracy but poor log-loss. - Prefer log-loss when: you care about calibration (probability outputs matter, e.g., medical risk); you're comparing models at the training/validation level.

Q10. What is the curse of dimensionality and how does it affect kNN?

A: In high-dimensional spaces (large d), data becomes extremely sparse. Key effects: 1. All pairwise distances become roughly equal as d→∞ (distance concentration). kNN can no longer distinguish "near" from "far" neighbors reliably. 2. Exponentially more data needed to maintain the same coverage density (sampling density falls as O(n^{-1/d})). 3. For kNN: distances become meaningless → neighbors are essentially random → classifier degrades toward random guessing. Fix: dimensionality reduction (PCA, t-SNE, UMAP), feature selection, or switch to a model that handles high-d better (SVM with kernel, tree ensembles, neural nets).

Q11. Explain dropout.

A: During training, randomly zero out each activation with probability p (dropout rate). The effective network varies per forward pass → acts like training an ensemble of 2^n subnetworks (one per possible mask pattern). At inference, scale activations by (1-p) or equivalently use inverted dropout (scale training activations by 1/(1-p) → no change at test time). Reduces co-adaptation of neurons. Improves generalization. Typical rates: 0.1–0.2 in Transformer attention layers; 0.5 for fully-connected layers in older architectures.

Q12. Why do we normalize features? What happens if we don't for gradient descent?

A: If features have very different scales (e.g., age in [0–100] and salary in [0–1,000,000]), the loss surface is elongated/elliptical. Gradient descent oscillates in the large-scale dimension while barely moving in the small-scale dimension. Learning rate that works for one feature blows up for another. Normalization (z-score or min-max) makes the loss surface more spherical → faster, more stable convergence. kNN and SVMs are also explicitly distance-based and require normalization.

Q13. What is the difference between parameters and hyperparameters?

A: - Parameters: learned from data during training. Example: weights W and bias b in a linear layer, embedding vectors. - Hyperparameters: set before training, not learned by gradient descent. Example: learning rate, batch size, number of layers, hidden dimension, dropout rate, k in kNN, C in SVM, n_estimators in Random Forest. - Hyperparameter tuning strategies: grid search (exhaustive), random search (usually better for same budget), Bayesian optimization (models p(metric|hyperparams)), population-based training.

Q14. Explain class imbalance and three ways to handle it.

A: When one class is far more frequent than others (e.g., 99% negative, 1% positive), models can achieve high accuracy by always predicting negative while being useless.

Solutions: 1. Resampling: oversample minority class (SMOTE = synthetic minority oversampling) or undersample majority class. Risk: overfitting with oversampling; information loss with undersampling. 2. Class weights: penalize misclassification of minority class more heavily during training (weight = n_majority/n_minority). Built into sklearn with class_weight='balanced'. 3. Threshold adjustment: move the classification threshold from 0.5 to lower value to increase recall for minority class. Then optimize threshold on validation precision-recall curve. 4. Bonus: use AUC-PR (precision-recall AUC) instead of AUC-ROC as evaluation metric — more informative when positive class is rare.

7I. Exam-Day Checklist

Before starting each coding question: 1. Read the full problem statement. Identify input type, output type, constraints (n=?). 2. Check edge cases: empty input, single element, all same, negatives, overflow risk. 3. Identify the pattern: array/hashmap? two-pointer? sliding window? tree? DP? 4. State your approach and complexity before coding (even silently). 5. Write the skeleton first (function signature, base cases) then fill in logic. 6. Test with the given example, then one edge case, before submitting.

Common bugs to avoid: - Off-by-one in binary search: while lo <= hi vs while lo < hi - Not copying when appending to result in backtracking: result.append(path[:]) not result.append(path) - Using list as default argument in function definition (it persists across calls): use None and init inside - Integer overflow: Python handles big ints automatically, but if the problem requires modular arithmetic, apply % MOD inside the loop - Forgetting to handle empty arrays/trees: always check if not arr or if not root - Modifying a list while iterating over it: iterate over a copy or use indices carefully

Complexity sanity checks: - n ≤ 10^6: O(n) or O(n log n) required - n ≤ 10^4: O(n²) acceptable - n ≤ 10^2: O(n³) or brute-force OK - n ≤ 20: exponential O(2^n) or O(n!) feasible
