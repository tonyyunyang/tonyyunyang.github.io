Unaided Practice Regimen

Your no-AI, no-autocomplete drill plan to rebuild raw coding muscle memory before Round 1 (Live Coding & Engineering). Start it 5–7 days out and run it daily. Open the Approach keys section only after a genuine attempt.

You're AI-native, so your real risk isn't knowing the algorithms — it's (a) typing them unaided and (b) narrating out loud while your hands move. This regimen drills exactly those two muscles. For the spoken UMPIRE protocol, the pattern catalogue, the ML-primitive reference code, and the recovery/hint playbook, see the companion doc Live Coding Mastery — this page is the training schedule that makes those automatic.

⚠ [CONFIRM TONY]: How many prep days and hours/day do you actually have? 7 days at ~2.5–3 h/day = full plan; compress to the 5-day track (below) if less.

Rules of engagement (non-negotiable)

  • AI off. Autocomplete off. Practice in a plain editor or a bare CoderPad-style pad — no linter fix-its, no Copilot ghost text. Your gap is unaided recall; simulate the real room.
  • Talk out loud, every rep. Record yourself or use a rubber duck. If you solved it silently, you did not practice the graded skill. Narrate all of UMPIRE, not just the code.
  • Timed, always. Every rep runs against a clock (cadence below). Time pressure is the point.
  • Type it, don't read it. Re-implementing a primitive from a blank file 3× beats re-reading it 10×. No copy-paste, ever.
  • Primitives from memory, daily. The ML shapes are the highest-leverage thing for this team and the fastest to make reflexive — one from-scratch primitive every single day.
  • No peeking before 25 min. Approach keys are at the very bottom for a reason. Earn them.

The per-problem rep cadence (the clock)

Every full rep is a mini interview round on one problem. Nest each problem into this budget and say the beat you're in out loud:

Beat Time What you do (out loud)
Talk / plan ~8 min Restate, clarify inputs/edge cases, name the pattern, state brute-force → improved approach and target complexity before typing. Get a nod.
Code ~25 min Narrate intent per block, meaningful names, guard clauses first. If stuck at ~25 min, read only the one-line key, close it, re-implement from memory.
Test ~10 min Hand-trace one normal case + the edge cases you named (empty, single, duplicates). Find and fix bugs out loud — that's a positive signal. Then state time + space complexity in three sentences.

Full-cadence rep = ~43 min. You won't do every problem at full cadence — that's a marathon. Each day:

  • 2 problems at full cadence (the 8/25/10 mock-round simulation, out loud, complexity stated).
  • Remaining problems as speed reps (~20 min each): still timed and narrated, but skip straight to the pattern and code — you're drilling recall, not the full ceremony.
  • 1 primitive from a blank file (~15 min), re-typed until it runs clean.

Day-by-day

Day 1 — Arrays / Hashmap + get the narration fluent

Pattern tell: "find a pair/complement," "count frequencies," "group by," "seen before?" → trade space for time with a dict / Counter / set. Problems (5): Two Sum · Group Anagrams · Top-K Frequent Elements · Valid Anagram · Contains Duplicate. Full cadence: Two Sum, Group Anagrams. Speed: the other three. Primitive from blank: Softmax (numerically stable) + Cross-Entropy. Nail the subtract-the-max trick and the +1e-12 in the log. Check your version against the ML-primitive reference code in Live Coding Mastery. Goal today: get the out-loud protocol comfortable on easy problems — the point of Day 1 is the mouth, not the difficulty.

Day 2 — Two Pointers + Sliding Window

Pattern tell: sorted input or in-place O(1) space → two pointers; "longest/shortest contiguous substring/subarray," "at most k distinct" → sliding window (expand right, shrink left on break). Problems (5): Valid Palindrome · 3Sum · Container With Most Water · Longest Substring Without Repeating Characters · Minimum Size Subarray Sum. Full cadence: 3Sum, Longest Substring No Repeat. Speed: the other three. Primitive from blank: kNN + z-score normalize. Talking point: normalize features before distances; O(N·D) per query.

Day 3 — Stack + Binary Search

Pattern tell: matching parens / "next greater" / monotonic stack; sorted array or "search the answer space" → binary search on a predicate. Problems (6): Valid Parentheses · Min Stack · Daily Temperatures · Binary Search · Search in Rotated Sorted Array · Koko Eating Bananas. Full cadence: Daily Temperatures, Koko Eating Bananas (search-on-answer is the hardest muscle here). Speed: the other four. Primitive from blank: Logistic Regression + gradient descent. The whole point: gradient of BCE is (p − y). Say it, don't derive it live.

Day 4 — Trees (BFS / DFS)

Pattern tell: binary/n-ary tree; "level order" → BFS with a deque; "depth / path / validate" → DFS recursion combining child results. Problems (6): Max Depth · Level-Order Traversal · Validate BST · Invert Binary Tree · Lowest Common Ancestor · Path Sum. Full cadence: Validate BST, Level-Order Traversal. Speed: the other four. Primitive from blank: Scaled Dot-Product Attention, then add a causal mask. Nail: divide by √d, softmax over the right axis, mask future with -1e9. Extend to multi-head verbally. Check against the reference code in Live Coding Mastery.

Day 5 — Graphs + Heaps

Pattern tell: grid/"islands"/connectivity → BFS/DFS flood-fill or union-find; prerequisites → Kahn's topo sort; "top-k / kth / merge k / running median" → heapq. Problems (5): Number of Islands · Course Schedule (topo sort) · Rotting Oranges (multi-source BFS) · Kth Largest Element · Merge k Sorted Lists. Full cadence: Course Schedule, Merge k Sorted Lists. Speed: the other three. Primitive from blank: K-Means + Precision/Recall/F1. Guard the empty-cluster case; guard every divide-by-zero in the metrics.

Day 6 — ML primitive gauntlet + BPE (the single most eBay-relevant session)

From a blank file, one sitting, implement all ten, then re-type any that stumbled until clean: softmax · cross-entropy · scaled dot-product attention · sigmoid + logreg · kNN · k-means · precision/recall/F1 · AUC (rank-based) · Welford running mean/var · BPE merge step. Then overfit logreg on a 4-point toy set to prove your code actually runs. BPE matters directly — eBay ships a custom e-commerce tokenizer; know that merge order matters (encoding replays merges in learned order). Reference shapes live in Live Coding Mastery. No speed/full split today — the whole day is the recall drill.

Day 7 — Two full mock rounds under time

  • Round A: one Tier-1 DSA problem, 45 min, out loud, timed, zero lookups.
  • Round B: one ML build-or-debug, 45 min. Debug bugs to expect: missing max-subtraction in softmax, wrong reduction axis, forgotten keepdims, off-by-one in a mask, sign error in a gradient.
  • Ideally with a friend who plants one deliberate hint so you rehearse taking it gracefully (paraphrase → integrate → extend past it).
  • Optional sampler if time: Subsets · Permutations · Coin Change · House Robber (backtracking / 1-D DP — state + recurrence out loud first).

Every-day fixed ribs (do these regardless)

  • One "complexity in three sentences" rep: time, then space (volunteer space unprompted), then the one tradeoff if it exists.
  • One "I'm stuck → recovery" rep: practice saying "Let me get the correct brute-force working first, then optimize" — securing a working baseline is the highest-value move you own.
  • Primitives never fully cool: even on DSA days, one primitive from blank keeps the shapes warm.

(The exact spoken scripts for narration, recovery, and hint-taking live in Live Coding Mastery — rehearse them there, drill them here.)

Compress to 5 days if short

  • Merge Day 3 into Day 2 (do 3 stack + 2 binary-search problems), and fold the Day 7 sampler into Day 6.
  • Keep Day 6 (gauntlet) and at least one full mock no matter what — they're non-negotiable.
  • Minimum viable week: Day 1, Day 2+3 merged, Day 4, Day 5, Day 6 (+ one mock). Primitives still daily.

Python power tools + gotchas (drill these to reflex)

Reaching for the right stdlib tool signals fluency; the gotchas below are common interviewer probes — be ready to explain each, not just avoid it.

Power tools

from collections import Counter, defaultdict, deque
Counter(s).most_common(k)          # frequency counts in one line; Counters add/subtract
adj = defaultdict(list); adj[u].append(v)   # kills the "key exists?" boilerplate
q = deque([root]); q.append(x); q.popleft() # O(1) both ends — the CORRECT BFS queue

import heapq
heapq.heappush(h, (priority, item)); heapq.heappop(h)   # min-heap; push -x for max-heap
heapq.nlargest(k, nums)            # top-k without hand-rolling a size-k heap

import bisect
i = bisect.bisect_left(a, x)       # O(log n) search in a sorted list
bisect.insort(a, x)                # keep a list sorted on insert

# comprehensions & generators — say "generator" when you don't need the whole list
squares = [x*x for x in nums if x > 0]
total   = sum(x*x for x in nums)   # generator: no intermediate list materialized

# enumerate / zip — never write range(len(...)) for indexing
for i, val in enumerate(nums): ...
for a, b in zip(xs, ys): ...
a, b = b, a + b                    # tuple unpacking / simultaneous assignment

The traps that bite (know cold)

Trap The bug The fix
Mutable default arg def f(x, acc=[]) — the list is created once at def time and shared across all calls, so it accumulates. def f(x, acc=None): acc = [] if acc is None else acc
Falsy-zero bug if x: is False for 0, 0.0, "", [], None. If 0 is valid data, you silently skip it. Test if x is not None: when 0 is a legal value.
Late-binding closure [lambda: i for i in range(3)] — all three capture the final i (=2). Bind now with a default: lambda i=i: i.
Shallow vs deep copy b = a aliases; a[:] / list(a) copies one level only. [[0]*n]*m shares every row. Nested → copy.deepcopy; grid → [[0]*n for _ in range(m)].
Integer division / is float; // floors toward −∞ (-7 // 2 == -4), not toward zero. Use // for indices, divmod(a, b) for both. Python ints are unbounded — no overflow (nice vs Java/C++).

Two more worth saying out loud: building a string with += in a loop is O(n²) — collect into a list and "".join(parts). And iterating a list/dict while mutating it misbehaves — iterate over a copy or build a new collection.

⚠ Approach keys — spoilers; don't read until you've tried

Read only after a genuine 25-minute attempt. These are one-line nudges to unstick, not full solutions.

Day 1 — Arrays / Hashmap

  • Two Sum: dict value→index; for each x check target−x in map before inserting x.
  • Group Anagrams: key by tuple(sorted(word)) (or 26-length count tuple) → defaultdict(list).
  • Top-K Frequent: Counter then heapq.nlargest(k, ...) — O(n log k); or bucket-sort by frequency for O(n).
  • Valid Anagram: length check, then compare Counter(a) == Counter(b) (or sorted).
  • Contains Duplicate: len(set(nums)) < len(nums), or seen-set with early exit.

Day 2 — Two Pointers / Sliding Window

  • Valid Palindrome: two pointers inward, skip non-alphanumerics, compare lowercased.
  • 3Sum: sort; fix i, two-pointer the rest; skip duplicates on all three indices.
  • Container With Most Water: two pointers at ends; move the shorter wall inward.
  • Longest Substring No Repeat: sliding window + last-seen-index map; jump left past the repeat.
  • Min Size Subarray Sum: grow right adding to sum; while sum ≥ target shrink left, record min length.

Day 3 — Stack / Binary Search

  • Valid Parentheses: stack; push opens, on close pop and match; empty stack at end = valid.
  • Min Stack: second stack of running minima (or store (val, min_so_far) tuples).
  • Daily Temperatures: monotonic decreasing stack of indices; pop when a warmer day arrives.
  • Binary Search: standard lo/hi; own the loop invariant, not just exact-match.
  • Search in Rotated: one half is always sorted — decide which, then whether target lies inside it.
  • Koko Bananas: binary-search the answer (eating speed); predicate = "can finish in h hours?"

Day 4 — Trees

  • Max Depth: DFS, 1 + max(left, right).
  • Level-Order: BFS with a deque, process one level (len(q)) at a time.
  • Validate BST: DFS carrying (low, high) bounds; each node strictly within.
  • Invert Tree: swap children, recurse.
  • LCA (BST): walk down; both values < node → left, both > → right, else you're at the split = LCA.
  • Path Sum: DFS subtracting node.val; at a leaf check remainder == 0.

Day 5 — Graphs / Heaps

  • Number of Islands: scan grid; on a 1, DFS/BFS flood-fill to sink the island, count once.
  • Course Schedule: build graph + indegrees; Kahn's BFS; cycle iff you can't process all nodes.
  • Rotting Oranges: multi-source BFS from all rotten cells at once; minutes = BFS levels.
  • Kth Largest: size-k min-heap; heap top is the answer.
  • Merge k Lists: min-heap of (val, i, node) across heads; pop smallest, push its next.

Day 7 — Backtracking / DP sampler

  • Subsets / Permutations: choose → recurse → undo; subsets branch include/exclude.
  • Coin Change / House Robber: 1-D DP; dp[i] = best up to amount/index i; state + recurrence before coding.

Primitive self-check cues (verify without opening the other doc)

  • Softmax: subtract row-max before exp; Cross-Entropy: −log(p[true]), add 1e-12.
  • Attention: scores = Q @ K.T / √d, mask future with -1e9, softmax over last axis.
  • Logreg: gradient of BCE is X.T @ (p − y) / N — beautifully (p − y).
  • kNN: normalize first, argsort(dists)[:k], majority vote. K-Means: guard the empty cluster.
  • PRF1 / AUC: guard every divide-by-zero; AUC = P(random positive ranked above random negative).
  • Welford: avoids catastrophic cancellation of "sum-of-squares − square-of-sum."
  • BPE: Counter adjacent pairs → merge the max-count pair → replay merges in learned order.
