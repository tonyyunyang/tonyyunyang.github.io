ML From Scratch (Python)

The ~10 NumPy primitives a Foundation-Models team asks you to type from a blank file — memorize these cold, ordered by likelihood. Read this the night before Round 1 and re-type each one from memory until it's automatic.

An ML-research round often swaps the LeetCode puzzle for "implement this in NumPy and overfit it on toy data" or "here's a broken model — find the bug." Fluency here should match your regular-Python fluency; fumbling the softmax formula for two minutes burns clock and signal. Narrate each one out loud the same way you would a DSA problem (the UMPIRE protocol from the Round-1 live-coding playbook applies unchanged).

Every entry below follows the same drill card: When asked (the trigger) → the ~10-line shape → Gotchas (say these aloud) → Complexity. All eight snippets are verified runnable.

import numpy as np
from collections import Counter

1. Softmax (numerically stable) + Cross-Entropy — #1 most likely

When asked: "Turn these logits into probabilities," or as a sub-step of attention / a classification loss. The subtract-the-max trick is the thing they're checking.

def softmax(x, axis=-1):                          # x: (..., C)
    x = x - np.max(x, axis=axis, keepdims=True)   # stability: prevents exp() overflow
    e = np.exp(x)
    return e / e.sum(axis=axis, keepdims=True)

def cross_entropy(logits, labels):                # logits (N,C), labels (N,) ints
    p = softmax(logits, axis=1)
    N = logits.shape[0]
    return -np.mean(np.log(p[np.arange(N), labels] + 1e-12))  # eps guards log(0)

Gotchas (say aloud):

  • Subtract the max before exp — softmax is shift-invariant, so it doesn't change the answer but stops exp(1000) overflowing to inf. This is the whole point of the question.
  • +1e-12 inside the log guards log(0) when a true-class probability underflows. And keepdims=True is what makes the divide broadcast correctly.
  • Stability, the right way: a stabler CE computes log_softmax(x) = x − logsumexp(x) directly and indexes that — never log(softmax(x)), which can still underflow. torch.nn.functional.cross_entropy fuses exactly this; the +1e-12 above is a band-aid for the naive form.
  • CE for one example = −log(prob of true class); averaging over the batch gives the loss.

Backward pass (the #1 follow-up): the gradient of CE wrt the logits is dL/dz = (p − y_onehot), divided by N for the mean — the same clean residual as binary BCE (one fact to remember).

grad = p.copy(); grad[np.arange(N), labels] -= 1; grad /= N

Complexity: O(N·C) time, O(N·C) space (one probability per logit).

2. Scaled Dot-Product Attention — #2, they're a Foundation-Models team

When asked: "Implement self-attention / a transformer block," or "add a causal mask." Expect a follow-up to extend to multi-head verbally.

def attention(Q, K, V, mask=None):                # Q:(L,d) K:(M,d) V:(M,dv)
    d = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d)                 # (L,M); scale keeps softmax gradients sane
    if mask is not None:                          # mask: True = keep, False = block
        scores = np.where(mask, scores, -1e9)     # causal mask hides future positions
    w = softmax(scores, axis=-1)                  # weights sum to 1 along each query row
    return w @ V                                  # (L,dv)

# causal mask for self-attention: mask = np.tril(np.ones((L, L), dtype=bool))

Gotchas (say aloud):

  • Divide by √d — without it, dot products grow with dimension, push softmax into a saturated corner, and gradients vanish. √d normalizes the variance back to ~1.
  • Causal mask is applied before softmax as -1e9 (≈ −∞ → weight 0 after exp), so a decoder position can't attend to future tokens. The mask is lower-triangular (tril); getting the triangle wrong silently leaks the future (see the debug section).

Multi-head, verbally: split d into h heads of size d/h, run this same attention independently per head, concat the h outputs back to d, then apply an output projection W_o. Same FLOPs as single-head — heads just let the model attend to different subspaces at once.

Multi-head, in code (if they push you past verbal):

def multihead(Q, K, V, h, W_o):                  # Q,K,V: (B, L, d)
    B, L, d = Q.shape; dh = d // h
    def split(x): return x.reshape(B, L, h, dh).transpose(0, 2, 1, 3)  # (B,h,L,dh)
    Qh, Kh, Vh = split(Q), split(K), split(V)
    scores = Qh @ Kh.transpose(0, 1, 3, 2) / np.sqrt(dh)   # (B,h,L,L)
    out = softmax(scores, axis=-1) @ Vh                     # (B,h,L,dh)
    out = out.transpose(0, 2, 1, 3).reshape(B, L, d)        # concat heads
    return out @ W_o                                        # output projection

Complexity: O(L·M·d) time and O(L·M) memory — quadratic in sequence length. That L² memory is exactly what FlashAttention (tiling, never materializing the full score matrix) exists to kill; worth naming for this team.

3. Precision / Recall / F1 + rank-based AUC

When asked: "Score this classifier by hand," "compute F1," or "implement AUC without sklearn."

def prf1(y_true, y_pred):                          # binary, 0/1
    tp = sum(t == 1 and p == 1 for t, p in zip(y_true, y_pred))
    fp = sum(t == 0 and p == 1 for t, p in zip(y_true, y_pred))
    fn = sum(t == 1 and p == 0 for t, p in zip(y_true, y_pred))
    prec = tp / (tp + fp) if tp + fp else 0.0      # guard divide-by-zero everywhere
    rec  = tp / (tp + fn) if tp + fn else 0.0
    f1   = 2 * prec * rec / (prec + rec) if prec + rec else 0.0
    return prec, rec, f1

def auc(y_true, scores):                           # rank-based (Mann–Whitney U)
    order = np.argsort(scores); ranks = np.empty(len(scores))
    ranks[order] = np.arange(1, len(scores) + 1)   # 1-based ranks by ascending score
    pos = np.array(y_true) == 1
    npos, nneg = pos.sum(), (~pos).sum()
    return (ranks[pos].sum() - npos * (npos + 1) / 2) / (npos * nneg)

Gotchas (say aloud):

  • Precision = "of what I predicted positive, how many were right"; recall = "of the actual positives, how many I caught"; F1 is their harmonic mean. Guard every denominator — no positives predicted → precision is 0/0.
  • AUC = P(a random positive is ranked above a random negative). The rank formula is the Mann–Whitney U identity; it's O(n log n), no threshold sweep needed. This snippet ignores exact score ties — mention you'd average tied ranks for exactness.

Complexity: PRF1 O(n) time, O(1) space. AUC O(n log n) time (the argsort), O(n) space.

4. BPE merge step — directly relevant: eBay ships a custom e-commerce tokenizer

When asked: "Implement one step of BPE training," or "how does subword tokenization learn its vocab?"

def get_stats(ids):                                # count every adjacent pair
    return Counter(zip(ids, ids[1:]))

def merge(ids, pair, new_id):                      # replace every occurrence of `pair`
    out, i = [], 0
    while i < len(ids):
        if i < len(ids) - 1 and (ids[i], ids[i + 1]) == pair:
            out.append(new_id); i += 2
        else:
            out.append(ids[i]); i += 1
    return out

# train loop: repeatedly pick the max-count pair, assign it a new id,
# merge it everywhere, and RECORD the merge in order.

Gotchas (say aloud):

  • Start from raw bytes (0–255), then greedily merge the single most-frequent adjacent pair each step, minting a new token id.
  • Merge order matters: encoding new text must replay the learned merges in the exact order they were learned — the recorded merge list is the model. Stop when you hit the target vocab size.

Complexity: O(n) per merge step (one pass over the sequence). Training to a vocab of size V is ~O(n·V) naive, before the standard priority-queue speedups.

5. Welford running mean/variance + z-score normalize — what LayerNorm does internally

When asked: "Compute mean/variance in one streaming pass," "standardize these features," or "what does LayerNorm compute?"

def running_mean_var(xs):                          # one pass, numerically stable
    n = 0; mean = 0.0; M2 = 0.0
    for x in xs:
        n += 1
        d = x - mean; mean += d / n                # update mean first...
        M2 += d * (x - mean)                        # ...then use BOTH old & new mean
    return mean, (M2 / n if n else 0.0)             # population variance (/n)

def zscore(X):                                      # per-feature standardization
    return (X - X.mean(0)) / (X.std(0) + 1e-8)      # eps avoids /0 on constant features

Gotchas (say aloud):

  • Welford avoids the catastrophic cancellation of the naive "mean of squares minus square of mean," which loses precision when values are large and variance is small.
  • Order matters inside the loop: update mean before the M2 term, which multiplies the old delta by the new delta. +1e-8 in z-score stops a constant feature (std 0) blowing up. This is exactly the normalize step inside LayerNorm/BatchNorm (they then rescale with learned γ, β).

Complexity: Welford O(n) time, O(1) space (its whole selling point). Z-score O(N·D).

6. k-Nearest Neighbors (kNN)

When asked: "Classify a point from its neighbors," the classic no-training warmup.

def knn_predict(X_train, y_train, x, k):           # X_train:(N,D), x:(D,)
    dists = np.linalg.norm(X_train - x, axis=1)     # Euclidean to every training point
    idx = np.argsort(dists)[:k]                     # indices of the k closest
    return Counter(y_train[idx]).most_common(1)[0][0]   # majority vote

Gotchas (say aloud):

  • No training — all the cost is at inference, so it doesn't scale to huge N without an ANN index (FAISS, HNSW).
  • Normalize features first — raw Euclidean distance is dominated by whichever feature has the largest scale. Pick odd k (or define a tie-break) to avoid split votes.

Complexity: O(N·D) per query, O(1) training. Add O(N log N) if you argsort the full distance array (a np.argpartition for top-k is O(N)).

7. k-Means (guard the empty cluster)

When asked: "Cluster this unlabeled data," Lloyd's algorithm from scratch.

def kmeans(X, k, iters=100):
    C = X[np.random.choice(len(X), k, replace=False)]     # init: k random points
    for _ in range(iters):
        d = np.linalg.norm(X[:, None] - C[None], axis=2)  # (N,k) all pairwise dists
        labels = d.argmin(1)                              # assign each point
        newC = np.array([X[labels == j].mean(0) if (labels == j).any()
                         else C[j] for j in range(k)])    # update; keep old if empty
        if np.allclose(newC, C): break                    # converged
        C = newC
    return labels, C

Gotchas (say aloud):

  • Guard the empty cluster — if no point is assigned to centroid j, .mean() is nan; keep the old centroid (or re-seed it). This is the edge case they're watching for.
  • Sensitive to init and only finds a local optimum — mention k-means++ seeding and running a few restarts. The X[:,None] - C[None] broadcast is O(N·k·D) memory, fine for toy data.

Complexity: O(iters·N·k·D) time.

8. Logistic Regression + Gradient Descent

When asked: "Train a binary classifier by hand," "derive/implement the gradient," then "overfit it on a 4-point toy set to prove it runs."

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

def train_logreg(X, y, lr=0.1, epochs=1000):       # X:(N,D), y:(N,) in {0,1}
    w = np.zeros(X.shape[1]); b = 0.0
    for _ in range(epochs):
        p = sigmoid(X @ w + b)
        gw = X.T @ (p - y) / len(y)                # gradient of BCE wrt w
        gb = (p - y).mean()                         # gradient wrt b
        w -= lr * gw; b -= lr * gb                  # descend (note the minus)
    return w, b

Gotchas (say aloud):

  • The BCE gradient collapses to a beautiful (p − y) — the prediction-minus-target residual, same shape as linear regression's. Know this cold; it's the thing they probe.
  • The update is w -= lr * grad (descend, not ascend — a sign flip here sends loss up). sigmoid overflows for large negative z; clip z if needed. Add L2 with gw += lam * w.

Complexity: O(epochs·N·D) time, O(D) space for the weights.

DEBUG THIS BROKEN MODEL

The other half of the round: they hand you working-looking code that's subtly wrong. Spot by symptom, not by re-reading line-by-line. These five account for most planted bugs.

Bug 1 — Missing max-subtraction in softmax

def softmax(x, axis=-1):
    e = np.exp(x)                     # BUG: no `x - max(x)`
    return e / e.sum(axis, keepdims=True)

Symptom: Works on toy data, then nan/inf at scale — the moment a logit gets large, exp overflows. Fix: subtract np.max(..., keepdims=True) first. This is the single most common planted bug for this team.

Bug 2 — Wrong reduction axis

p = e / e.sum(axis=0, keepdims=True)   # BUG: normalizing down the batch, not across classes

Symptom: Probabilities sum to 1 across the wrong dimension (over examples instead of over classes), so per-row probs don't sum to 1 and loss is nonsense. Sanity check: softmax(logits).sum(axis=1) must be all-ones for (N, C) logits.

Bug 3 — Missing keepdims

x = x - np.max(x, axis=-1)             # BUG: no keepdims → shape (N,) not (N,1)

Symptom: Either a broadcasting ValueError, or worse, it broadcasts against the wrong axis and silently produces garbage. Whenever you subtract/divide a reduction from its source array, keepdims=True.

Bug 4 — Off-by-one causal mask (the dangerous one)

mask = np.tril(np.ones((L, L), bool), k=1)   # BUG: k=1 lets each token see ONE future token

Symptom: No crash, trains beautifully, cheats at inference. k=1 leaks one future token (label leakage); k=-1 blocks a token from attending to itself. The correct causal mask is plain np.tril(...) (k=0). Suspect this whenever training loss looks too good — silent future-leakage is the classic transformer footgun.

Bug 5 — Gradient sign error

w += lr * gw                            # BUG: ascending the gradient
# or:  gw = X.T @ (y - p) / len(y)      # BUG: residual flipped to (y - p) then subtracted

Symptom: Loss goes up every epoch (or oscillates/diverges). Descent is w -= lr * grad with residual (p - y). If you flip the residual to (y - p) you must flip the update to +=; get exactly one of the two wrong and it diverges. First move on any diverging trainer: print the loss each epoch and check the sign.

30-second pre-flight checklist (run this mentally before you say "done")

  • Softmax: max subtracted? keepdims? right axis? log(0) guarded with eps?
  • Attention: ÷√d? mask applied before softmax? causal triangle is tril(k=0)?
  • Any metric: every denominator guarded against zero?
  • Any trainer: update is -= lr * grad, residual is (p - y), loss decreasing in a quick trace?
  • k-means: empty cluster handled? kNN: features normalized?
