A trace-driven framework that treats a single model invocation as the unit of adaptation and evolves three components over shared agent traces: a SkillBook of statistics for recurring prompt signatures, a learned invocation-level router, and a small-model adapter. At runtime an input-only router picks among strong, cheap, and specialist models, a skill layer dispatches stable templates for recurring local structure, and a verifier protects quality through fallback. Updates are admitted only when joint replay shows the combined policy still satisfies verification constraints. In a code-generation setting with 590 executable tasks and 3,328 weakly labelled router examples, the best four-cycle order is Skill → LLM → Router, reaching 87.3% router accuracy with 4.4% fallback and cutting estimated serving cost to 51.8% of always using the large model.
